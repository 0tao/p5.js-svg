loadGraphics(graphics, successCallbackopt, failureCallbackopt)

Due to a known issue (image is not surely ready before onload fires), we have no way to draw SVG element synchronously. So, this method will load a SVG Graphics and then convert it to Canvas Graphics asynchronously
Parameters:
Name Type Attributes Description
graphics p5.Graphics the p5.Grphaics object
successCallback function <optional>
 Function to be called once the SVG Graphics is loaded. Will be passed the p5.Graphics.
failureCallback function <optional>
 called with event error.
Source:
  • rendering.js, line 24
See:
  • https://github.com/zenozeng/p5.js-svg/issues/78
Example
<div>
<code>
pg = createGraphics(100, 100, SVG);
background(200);
pg.background(100);
pg.ellipse(pg.width/2, pg.height/2, 50, 50);
loadGraphics(pg, function(pgCanvas) {
     image(pgCanvas, 50, 50);
     image(pgCanvas, 0, 0, 50, 50);
});
</code>

saveSVG(svg, filename, extension)

Save the current SVG as an image. In Safari, will open the image in the window and the user must provide their own filename on save-as. Other browsers will either save the file immediately, or prompt the user with a dialogue window.
Parameters:
Name Type Description
svg Graphics | SVGElement Source to save (optional)
